Mini mini mini glossary on Agents


AGENTS: 

Agent
An agent uses model-reasoning to plan, execute actions, observe results and adapt its approach over multiple turns using the observe-think-act loop and "tool calling" to get more data into the loop. 

Sub-Agent
A child agent spawned by a parent agent into a fresh context window, enabling memory isolation for a bounded unit of work, bounded tasks and irrelevant intermediate states.

AGENTS.md
An open&standard vendor-neutral file format proposed by Geoffrey Huntley in 06/2025 for instructions to AI coding agents, now governed by Agentic AI Foundation (AAIF) under the Linux Foundation.

CLAUDE.md
is a Claude specific variant of AGENTS.md which is automatically to context and includes persistent, project-specific instructions, giving the agent immediate context about the codebase’s architecture, build commands, testing procedures and coding conventions.

Agent Harness
Harness is the orchestration layer around an agent that constructs context, assembles the system prompt, reads the agent.md, executes "tool calls", enforces guardrails and decides when to loop - sort of an operating system for the agent. 

Model Context Protocol (MCP)
Open standard by Anthropic that defines how agents connect to external sources in a client-server architecture to discover and invoke resources on behalf of the language model, loaded in the context in a similar way as static linking libraries in software.

Agent Skills
Instruction sets that an agent lazy-loads into its context window only when it determines they are relevant to the current task, opposite of MCP.

Agent Heartbeat
A liveness detection mechanism where an agent periodically updates a timestamp in a shared database, allowing a monitoring system to detect crashed agents and automatically reassign their uncompleted work.

Tool Calling
A mechanism that allows large language models to query a database, call a web API or run a calculation — and feed the result back to the model as a new message.

Context window
A fixed-size buffer of tokens that an LLM can “see” at once, including everything the model reads (the system prompt, conversation history, tool definitions, and tool results) plus everything it writes (the response text and any tool calls).

Compaction
A “garbage collection” strategy that removes older messages from an agent’s conversation to free space in the context, usually dangerous for hallucinacions and mitigated by pinning critical allocations (system prompt, specs, .md), spawning subagents for fresh contexts or Ralph loops with persistence in between.

Context engineering
The context window is analogous to RAM and we design, manage and otpimize the information in it to optimise the reliability and quality of model's output with allocations, deallocations, caring for fixed allocations, memory isolation (subagents) and context budgeting.

Finite-State Machine (FSM)
A way to enforce sequences of operations in an agent loop my modeling which transitions are legal and preventing the system from entering invalid states — like executing a tool while already executing one, or producing a response before processing a tool result, primarily developed as software code to maintain determinism.

Ralph Wiggum Loop
A pattern for iteration over perfection, where work is split into small iterations, and each iteration runs in a fresh context window instead of keeping one long chat alive until it degrades - while true; do - used with backpressure and subagents.

Agent backpressure
The practice of surrounding an agent with automated like type checkers, test suites, linters and pre-commit hooks — that allow AI agents to detect and correct their own mistakes without human intervention, sometimes implemented in code using tools like LangGraph.

Steering Message
A user message injected into an agent's conversation mid-run that interrupts remaining tool executions and redirects the agent's behavior, typicallly between tool calls.

Follow-up Message
Unlike steering messages, these are user messages queued and delivered after the agent completes all of its tool calls and produces a response, then triggering a new turn.

Provider Registry
Basically a config file to decouple from the LLM provider and to be able to change LLMs from e.g. Anthropic to Mistral, including mid conversation, enabling multi-model agents.

Swarm Analysis
At planning time before executing a large epic, swarm analysis goes through the graph of work items and groups them into waves — sets of tasks that share the same depth level and can execute concurrently in waves, waiting for eachother.



MODELS:

Token and Tokenizers
The tokenizer is the interface between human-readable text and the model’s numerical world which converts text into a sequence, where most modern GPT models use subword tokenization, meaning converting "hello" to a token group "hel, lo".


Streaming
A technique where a language model sends its response incrementally as it is generated, rather than waiting for the complete output, allowing for early cancellation if the model is on a wrong path.

Inference
The process of running a trained model on new inputs to produce predictions, while not updating weights and being stateless using context window as the only memory.

System prompt
A system prompt is a fixed allocation message placed as the first allocation in a LLM's context window (followed by harness prommpt, AGENTS.md and MCP), defining the model’s behavior, personality, capabilities, and constraints for the conversation and is treated with higher trust by most models.

Prompt injection
An adversarial attack in which a user crafts input that overrides or manipulates a language model’s system instructions, causing it to behave in unintended ways.

Transformer
A neural network architecture based on self-attention mechanisms, introduced in 'Attention Is All You Need'. They are the foundation of LLMs, used for machine translation, text summarization and image analysis.

GPT - Generative Pre-trained Transformer 
A family of decoder-only language models, trained on large text corpora using a simple objective: predict the next token in a sequence.

Temperature (Sampling)
Temperature is a parameter that controls how random a language model’s output is during text generation, where low temperature produces more deterministic outputs and higher temperature produceds more varied and suprising (creative) text, meaning that lower-probability tokens get more chance.
